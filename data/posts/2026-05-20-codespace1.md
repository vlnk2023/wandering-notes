**利用 GitHub Codespaces 搭建干净 IP 节点的实用方法**（适合低强度、临时使用）。

Codespaces 的出口 IP 来自 **GitHub/Microsoft 的基础设施**，属于数据中心 IP，但相对“干净”：不是廉价机房/滥用 NAT 池，历史记录好，黑名单风险低，常用于临时代理/解锁场景。IP 是动态的，每次新建可能不同，支持选 region（影响 IP 归属地）。

**免费额度**：个人免费约 120 core-hours/月（2核配置大概 60 小时），用完要付费或停掉 Codespace。

### 推荐最简单方式（Fork 现成项目）
1. **g2ray 项目**（支持 v2ray/Xray 等协议）：
   - 去 https://github.com/amircloner/g2ray Fork 到自己账号。
   - 打开 Fork 仓库 → 点击 **Code** → **Codespaces** → **Create codespace on main**。
   - 等待构建（几分钟），终端会输出连接字符串（vmess/vless 等）。
   - 在客户端（如 v2rayNG、Nekobox、Clash）导入即可使用。

2. **SOCKS5 Proxy 项目**（用 Chisel）：
   - Fork https://github.com/AnimMouse/SOCKS5-Proxy-Codespaces。
   - 新建 Codespace，根据 README 操作：
     - 暴露端口 8080 为 Public。
     - 本地运行 `chisel client <端口地址> socks`。
     - 浏览器/系统设本地 SOCKS5（127.0.0.1:1080）。支持 Cloudflare Tunnel 加速。

3. **手动 Squid HTTP 代理**（简单但仅 HTTP/HTTPS）：
   - 新建任意仓库，打开 Codespace。
   - 创建 `install.sh` 脚本安装 Squid + ngrok 转发端口。
   - 加到 `.bashrc` 自动启动。
   - 用 ngrok 暴露后，本地配置 HTTP 代理。

### 提升纯净度 & 防封技巧
- **新建时选 Region**：不同地区 IP 归属地不同，测试哪个好用（欧美 IP 通常更干净）。
- **结合 CDN/中转**：把 Codespaces 作为源站，用 Cloudflare 回源，只暴露 CDN IP，进一步隐藏。
- **低调使用**：别高频滥用、别跑代理池。每次用完 **Stop** 或删除 Codespace，养 IP。
- **检测 IP**：用 `ping0.cc/ip`、`ipinfo.io`、`whoer.net` 检查风险值、是否住宅-like、黑名单。
- **协议混淆**：用 Reality、gRPC、WS+TLS 等抗封协议。
- **多开**：同时开几个不同 region 的 Codespace 切换。

### 注意事项 & 限制
- **不适合高强度/长期**：免费时长有限，GitHub 会监控滥用（高流量、恶意行为可能封账号）。
- **端口转发**：Codespaces 默认只支持有限端口公开，SOCKS/VMess 需 Chisel/ngrok/cloudflared 隧道。
- **IP 动态**：重启/新建会换 IP，优点是“新鲜”，缺点是不固定。
- **合规**：仅用于学习/个人合法用途，遵守 GitHub ToS 和当地法规。
